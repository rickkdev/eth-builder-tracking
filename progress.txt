## Codebase Patterns
- Foundry project lives in `contracts/` with OpenZeppelin v5.6.1 and forge-std v1.15.0
- Remappings: `@openzeppelin/contracts/` maps to `lib/openzeppelin-contracts/contracts/`
- CodesRegistry uses custom errors (not require strings) for gas efficiency
- Token IDs are sequential starting from 0 via `_nextTokenId++`
- Code lookups use `keccak256(bytes(code))` as mapping key for O(1) access
- Forge linter notes about plain imports and asm-keccak256 are informational only, not errors
- Foundry scripts go in `contracts/script/`, use `vm.startBroadcast()`/`vm.stopBroadcast()`, run with `forge script` + `--broadcast`
- Anvil default account #0 key: `0xac0974bec39a17e36ba4a6b4d238ff944bacb478cbed5efcae784d7bf4f2ff80`
- SE2 frontend lives in `apps/web/` with Next.js 15, React 19, Wagmi 2.19.5, Viem 2.47.4
- ox/erc8021 `Attribution.toDataSuffix({ codes: ['code'] })` generates ERC-8021 suffix (note: `codes` is plural array)
- SE2 uses `~~/*` path alias for root imports, DaisyUI 5 + Tailwind 4 for styling
- Contract ABI goes in `apps/web/contracts/deployedContracts.ts` keyed by chainId
- SE2 scaffold hooks: `useScaffoldReadContract`, `useScaffoldWriteContract`, `useScaffoldEventHistory`
- Use `--legacy-peer-deps` for npm install (React 19 peer dep conflicts with some packages)
- Nav links configured in `apps/web/components/Header.tsx` `menuLinks` array
- SE2 scaffold hooks expect args as `[valueOrUndefined]` tuple, not `undefined` for the entire array
- Use `getBlockExplorerTxLink(chainId, txHash)` from `~~/utils/scaffold-eth/networks` for Etherscan links
- viem `publicClient.multicall()` uses Multicall3 automatically — pass array of `{ address, abi, functionName, args }` contracts
- Always run `npx prettier --write` on new page files before committing
- ox/erc8021 `Attribution.fromData(txInput)` decodes ERC-8021 suffixes; returns `{ codes: string[] }` or `undefined`
- Next.js 15: `useSearchParams()` requires `<Suspense>` boundary — extract inner component, wrap in page-level Suspense
- For user-provided external image URLs, use `<img>` with `eslint-disable-next-line @next/next/no-img-element`
